# Design: Internacionalização (i18n) — Português / Inglês

**Data:** 2026-03-21
**Status:** Aprovado
**Branch:** feature/redesign-visual-sagrado

---

## Contexto

O site da Casa da Alquimia (React + TypeScript + Vite + react-router-dom v6) possui todo o conteúdo em português hardcoded nos componentes. O objetivo é adicionar suporte ao idioma inglês, com seletor na navbar, rota compartilhável `/en` e detecção automática pelo idioma do browser.

---

## Decisões

| Decisão | Escolha |
|---------|---------|
| Biblioteca i18n | react-i18next |
| Estratégia de rota | Prefixo `/en` para inglês, `/` para português |
| Seletor na navbar | Dropdown com ícone de globo (🌐) |
| Detecção automática | `navigator.language` (sem API externa) |
| Persistência | `localStorage` (`lang` key) |
| Traduções | Criadas junto à implementação, baseadas no conteúdo PT |
| Tom do inglês | Acessível e próximo ao português, sem rebuscamento |

---

## Arquitetura

### Estrutura de arquivos novos

```
src/
  i18n/
    index.ts          ← configuração do react-i18next
    pt.json           ← todas as strings em português
    en.json           ← todas as strings em inglês
```

### Rotas

```
/                           → Index (PT)
/en                         → Index (EN)
/politica_de_privacidade    → PoliticaDePrivacidade (PT)
/en/politica_de_privacidade → PoliticaDePrivacidade (EN)
*                           → NotFound
```

A lógica de idioma é determinada pela URL, não por estado global. Ao detectar `/en` no path, a instância do i18next muda para `en`; caso contrário, usa `pt`.

### Fluxo de detecção automática (primeiro acesso)

```
Usuário acessa qualquer URL
  ↓
Path contém /en?  →  sim  →  idioma = EN (salva no localStorage)
  ↓ não
localStorage['lang'] existe?  →  sim  →  redireciona para / ou /en conforme valor salvo
  ↓ não
navigator.language começa com 'pt'?  →  sim  →  idioma = PT, permanece em /
  ↓ não
Redireciona para /en
```

## Seletor de Idioma (Navbar)

**Estilo escolhido:** Dropdown com globo (Opção C)

- Ícone de globo (🌐 ou `Globe` do lucide-react) + nome do idioma atual + chevron
- Clique abre dropdown com as duas opções (PT 🇧🇷 / EN 🇺🇸)
- Item ativo recebe checkmark
- Ao selecionar: navega para `/` ou `/en` preservando o path relativo (ex: `/politica_de_privacidade` → `/en/politica_de_privacidade`)
- Fecha ao clicar fora
- No mobile: também aparece no menu lateral

## Comportamento das Rotas

### App.tsx — novas rotas

```tsx
<Routes>
  <Route path="/" element={<Index />} />
  <Route path="/en" element={<Index />} />
  <Route path="/politica_de_privacidade" element={<PoliticaDePrivacidade />} />
  <Route path="/en/politica_de_privacidade" element={<PoliticaDePrivacidade />} />
  <Route path="*" element={<NotFound />} />
</Routes>
```

### useLanguageRoute hook

Executado no App ou num componente wrapper:
1. Lê o path atual
2. Determina idioma correto (`pt` ou `en`)
3. Chama `i18n.changeLanguage(lang)`
4. Persiste em `localStorage`
5. Na primeira visita sem preferência salva: redireciona se `navigator.language` não for PT

## Setup do react-i18next (`src/i18n/index.ts`)

```ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import pt from './pt.json';
import en from './en.json';

i18n.use(initReactI18next).init({
  resources: { pt: { translation: pt }, en: { translation: en } },
  lng: 'pt',
  fallbackLng: 'pt',
  interpolation: { escapeValue: false },
});

export default i18n;
```

## Uso nos Componentes

```tsx
import { useTranslation } from 'react-i18next';

const Hero = () => {
  const { t } = useTranslation();
  return <h1>{t('hero.title')}</h1>;
};
```

## Critérios de Aceitação

- [ ] Acessar `/en` mostra o site em inglês
- [ ] Acessar `/` mostra o site em português
- [ ] Seletor na navbar troca idioma e navega para rota correspondente
- [ ] Seletor aparece no menu mobile também
- [ ] Preferência de idioma persiste ao recarregar (localStorage)
- [ ] Usuário com browser não-PT é redirecionado para `/en` na primeira visita
- [ ] URL `/en` pode ser compartilhada e abre em inglês diretamente
- [ ] Links internos (ex: Footer, PoliticaDePrivacidade) preservam o idioma ativo
- [ ] Build passa sem erros de TypeScript
