# Design: Internacionalização (i18n) — Português / Inglês

**Data:** 2026-03-21
**Status:** Aprovado
**Branch:** feature/redesign-visual-sagrado

---

## Contexto

O site da Casa da Alquimia (React + TypeScript + Vite + react-router-dom v6) possui todo o conteúdo em português hardcoded nos componentes. O objetivo é adicionar suporte ao idioma inglês, com seletor na navbar, rota compartilhável `/en` e detecção automática pelo idioma do browser.

---

## Decisões

| Decisão | Escolha |
|---------|---------|
| Biblioteca i18n | react-i18next |
| Estratégia de rota | Prefixo `/en` para inglês, `/` para português |
| Seletor na navbar | Dropdown com ícone de globo (🌐) |
| Detecção automática | `navigator.language` (sem API externa) |
| Persistência | `localStorage` (`lang` key) |
| Traduções | Criadas junto à implementação, baseadas no conteúdo PT |
| Tom do inglês | Acessível e próximo ao português, sem rebuscamento |

## Arquitetura

### Estrutura de arquivos novos

```
src/
  i18n/
    index.ts          ← configuração do react-i18next
    pt.json           ← todas as strings em português
    en.json           ← todas as strings em inglês
```

### Rotas

```
/                           → Index (PT)
/en                         → Index (EN)
/politica_de_privacidade    → PoliticaDePrivacidade (PT)
/en/politica_de_privacidade → PoliticaDePrivacidade (EN)
*                           → NotFound
```

A lógica de idioma é determinada pela URL, não por estado global. Ao detectar `/en` no path, a instância do i18next muda para `en`; caso contrário, usa `pt`.

### Fluxo de detecção automática

**Regra principal: a URL sempre tem prioridade sobre o localStorage.**

Se o usuário acessou uma URL explícita com ou sem `/en`, esse idioma é adotado e salvo — independente do que estiver no localStorage. O localStorage só é consultado quando não há prefixo de idioma inferível da URL (o que na prática nunca acontece com esse modelo de rotas).

```
Usuário acessa qualquer URL
  ↓
Path contém /en?  →  sim  →  idioma = EN, salva "en" no localStorage
  ↓ não
É a primeira visita? (localStorage['lang'] vazio)
  →  sim: navigator.language começa com 'pt'?
        →  sim  →  idioma = PT, permanece em /
        →  não  →  redireciona para /en, salva "en" no localStorage
  →  não: idioma = PT, permanece em / (URL explícita sem /en tem prioridade)
```

**Comportamento chave:** acessar `/` manualmente sempre carrega PT, mesmo que o localStorage tenha "en" salvo de uma visita anterior. O usuário que digita a URL escolhe explicitamente o idioma.

**O que o localStorage persiste:** apenas a preferência para o fluxo de detecção automática na *primeira visita* (sem path explícito). Em visitas subsequentes, a URL continua sendo a fonte de verdade — o localStorage não redireciona automaticamente. A critério de aceite "Preferência de idioma persiste ao recarregar" significa: ao recarregar estando em `/en`, o site continua em inglês (o que é garantido pela própria URL, não pelo localStorage).

## Seletor de Idioma (Navbar)

**Estilo escolhido:** Dropdown com globo (Opção C)

- Ícone de globo (🌐 ou `Globe` do lucide-react) + nome do idioma atual + chevron
- Clique abre dropdown com as duas opções (PT 🇧🇷 / EN 🇺🇸)
- Item ativo recebe checkmark
- Ao selecionar: navega para `/` ou `/en` preservando o path relativo (ex: `/politica_de_privacidade` → `/en/politica_de_privacidade`)
- Se o path atual não é uma rota conhecida (ex: usuário está em `*` NotFound), cai para `/` (PT) ou `/en` (EN)
- Fecha ao clicar fora
- No mobile: também aparece no menu lateral

## Comportamento das Rotas

### App.tsx — novas rotas

```tsx
<Routes>
  <Route path="/" element={<Index />} />
  <Route path="/en" element={<Index />} />
  <Route path="/politica_de_privacidade" element={<PoliticaDePrivacidade />} />
  <Route path="/en/politica_de_privacidade" element={<PoliticaDePrivacidade />} />
  <Route path="*" element={<NotFound />} />
</Routes>
```

### useLanguageRoute hook

Executado em `App.tsx`, dentro de `<BrowserRouter>` mas fora de `<Routes>`, via um componente wrapper `<LanguageRouter>`. Isso garante acesso ao `useLocation()` do react-router e aplica o idioma antes de qualquer rota renderizar.

```tsx
// App.tsx
<BrowserRouter>
  <LanguageRouter>   {/* ← hook roda aqui */}
    <Routes>...</Routes>
  </LanguageRouter>
</BrowserRouter>
```

Responsabilidades:
1. Lê o path atual via `useLocation()`
2. Determina idioma: path começa com `/en` → `en`, caso contrário → `pt`
3. Chama `i18n.changeLanguage(lang)`
4. Persiste em `localStorage` (ver regras de persistência abaixo)
5. Na primeira visita sem preferência salva: redireciona via `useNavigate()` se `navigator.language` não for PT

## Links Internos Cientes de Idioma

Todos os links internos (Footer, Navbar) usam o hook `useLocalizedPath`:

```tsx
import { useLocalizedPath } from '@/hooks/useLocalizedPath';

const Footer = () => {
  const lp = useLocalizedPath();
  return <Link to={lp('/politica_de_privacidade')}>Política de Privacidade</Link>;
};
```

O hook recebe um path raiz e retorna o path com o prefixo `/en` quando o idioma ativo é inglês:

```ts
// useLocalizedPath.ts
export const useLocalizedPath = () => {
  const { i18n } = useTranslation();
  return (path: string) => i18n.language === 'en' ? `/en${path}` : path;
};
```

**Componentes que usam links internos e devem adotar `useLocalizedPath`:** Footer, Navbar (link do logo), NotFound (link "voltar para início").

**NotFound em múltiplos idiomas:** A rota `*` renderiza o `NotFound` — o `LanguageRouter` também executa nessa rota, portanto o idioma é detectado pelo path e `NotFound` exibe o conteúdo no idioma correto via `useTranslation`.

## Setup do react-i18next (`src/i18n/index.ts`)

```ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import pt from './pt.json';
import en from './en.json';

i18n.use(initReactI18next).init({
  resources: { pt: { translation: pt }, en: { translation: en } },
  lng: 'pt',
  fallbackLng: 'pt',
  interpolation: { escapeValue: false },
});

export default i18n;
```

import { useTranslation } from 'react-i18next';
